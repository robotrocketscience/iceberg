**Authorial reading:**

The audit's value to the campaign is not in finding new live bugs (none found). It is in three structural deliverables:

1. **Diagnosis reframe**: lesson 6's "shared-library function input conventions need assertions" mis-locates the bug. The flagged seed-list functions (`constant_thrust_burn`, `edelbaum_spiral`, `spiral_burn`) are NOT in the shared library — they are copy-pasted into 6 rounds with one convention-inverted variant. The actual contagion is round-local redefinition with per-round drift. Lesson 6 should be amended (or a new lesson added) to capture this.

2. **Canonical helper-pair pattern**: `R_electric_outbound_rerun`'s `burn_from_wet` / `burn_from_dry_end` pattern is lifted into the shared library at `waterprop.propulsion.burns`. Future rounds importing the canonical pair commit to a convention by the function name, removing the silent-bug class.

3. **Enforcement gate**: `audit-call-conventions.py` runs in 2.2 seconds and exits 1 if any round-local redefinition of `constant_thrust_burn` (or any other shared-helper-canonical-name) is detected, AND if any caller passes a dry-hint-named variable to `burn_from_wet`. The gate is conservative — false positives are acceptable in exchange for catching the bug class — and false positives are documented as carve-outs in `audit_carveouts.py`.

The cleanup is finite (3 rounds), the prevention mechanism is cheap (132 lines, 2.2s), and the architectural decisions remain where latest+13 left them. The "every test is contaminated" framing is overstated by an order of magnitude; the actual contaminated count is 3 out of 136.
